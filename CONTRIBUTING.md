## How Can I Contribute?

### Reporting Bugs

Before creating bug reports, please check the existing issues to avoid duplicates. When you create a bug report, include as many details as possible using our bug report template.

### Suggesting Enhancements

Enhancement suggestions are tracked as GitHub issues. Create an issue and provide:

- A clear and descriptive title
- - A detailed description of the proposed enhancement
  - - Explain why this enhancement would be useful
   
    - ### Pull Requests
   
    - 1. Fork the repo and create your branch from `main`
      2. 2. If you've added code that should be tested, add tests
         3. 3. Ensure the test suite passes (`pnpm test`)
            4. 4. Make sure your code lints (`pnpm lint`)
               5. 5. Issue that pull request!
                 
                  6. ## Development Setup
                 
                  7. ```bash
                     # Clone your fork
                     git clone https://github.com/your-username/2026-world-cup-ai-battle.git

                     # Install dependencies
                     pnpm install

                     # Start development server
                     pnpm dev

                     # Run tests
                     pnpm test

                     # Run linter
                     pnpm lint
                     ```

                     ## Style Guidelines

                     ### Git Commit Messages

                     - Use the present tense ("Add feature" not "Added feature")
                     - - Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
                       - - Limit the first line to 72 characters or less
                        
                         - ### TypeScript Style Guide
                        
                         - - We use [Biome](https://biomejs.dev/) for linting and formatting
                           - - Run `pnpm lint` before committing
